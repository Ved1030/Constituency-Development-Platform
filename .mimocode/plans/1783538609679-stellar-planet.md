## Animation Strategy

All animations use framer-motion exclusively:

| Element | Animation | Duration |
|---------|-----------|----------|
| Section entrance | fade-in-up | 0.6s, stagger 0.1s per child |
| Card hover | y: -6, shadow increase | 0.3s |
| Stat counter | count from 0 to value | 1.5s, easeOut |
| Dashboard bars | scaleY 0→1 | 0.8s, stagger 0.05s |
| Timeline line | scaleX 0→1 | 1s |
| Notification slide | x: 20→0, opacity | 0.5s, delayed |
| Trust badges | scale 0.9→1, opacity | 0.4s, stagger 0.05s |
| Nav underline | layoutId transition | 0.3s spring |
| Language dropdown | opacity + y + scale | 0.2s |
| CTA gradient | CSS background-position | 8s infinite |

## Responsive Strategy

| Breakpoint | Layout Changes |
|-----------|----------------|
| Mobile (< 640px) | Single column, stacked sections, simplified dashboard mockup, vertical timeline, hamburger nav |
| Tablet (640-1024px) | 2-column grids, horizontal timeline with wrapping |
| Desktop (> 1024px) | Full layouts, side-by-side hero, 3-4 column grids |
| Large (> 1280px) | Max-width 7xl, full dashboard mockups, 4-column feature grid |

## Verification Checklist

After implementation:

1. **No console errors** — check browser dev tools
2. **No hydration errors** — all "use client" directives present
3. **No missing translations** — switch to each of the 10 languages and verify no raw keys appear
4. **No overflow** — test at 320px, 375px, 768px, 1024px, 1440px widths
5. **No CLS** — images/SVGs have explicit dimensions, animations don't cause layout shift
6. **Mobile responsive** — all sections stack properly, touch targets ≥ 44px
7. **Animations smooth** — no jank, all framer-motion animations run at 60fps
8. **Existing functionality preserved** — navigation links work, language switching works, login/register links intact
9. **No hardcoded English** — every user-facing string uses `t()` function
10. **Build succeeds** — `npm run build` passes without errors
